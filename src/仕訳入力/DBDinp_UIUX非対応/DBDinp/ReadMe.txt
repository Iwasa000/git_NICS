SEQ_NO			最初に ゼロ をセットして、追加仕訳があるたびに　＋１ する。
				その番号の位置に、追加分の仕訳番号をセットして行く。

BOTOM_SEQ		修正仕訳リストの底データのSEQ。 0の場合は、修正リストに表示データなし。


REAL_SEQ_NO		仕訳が増えると、マルチで実際にセットされた番号をセットしていく


SCAN_RD_BOTOM	仕訳検索修正中の INP_VIEW で最下行に表示している
				データの SCAN_VIEW 画面での オフセット

DATA_1RD ... 0 の時は、CREC に読み込まれていないので、データを読み込む
			 1 の時は、CREC に現ラインのデータが読み込まれている。
			 1 の時のみ、データを残高に書く( rewupdat() をする )


reupdat(&CREC) ....	引数 CREC より SCAN_rec の グローバル変数にコピー
					ファイルよりCREC を読み込んだら 必ずする
					CREC は 変更格納、SCAN_rec は CREC を書き込む時の変更点比較のための
					元の仕訳を格納する変数
